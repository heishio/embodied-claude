# 記憶システム 

実装済み概念

---

## 1. 意味は経路

体験を**動詞チェーン**で記録する理由。
「猫を撫でて安心した」と「犬を撫でて安心した」。名詞は違うけど、人間は「似てる」と感じる。似てるのは猫と犬じゃなくて、「撫でる → 安心する」という流れの方。

```
見る(空) → 気になる(色) → 調べる(天気) → 理解する(理由)
```

ベクトル化するとき、2つの軸に分離する：

- **flow_vector**: 動詞バイグラム（隣接動詞ペアの中点）の平均。「何をしたか」
- **delta_vector**: 名詞平均 − 動詞平均。「何に対してか」

chiVe（日本語word2vec, 300次元）で各単語をベクトル化し、上記の計算で2本のベクトルを得る。検索時は2軸を独立に比較する（flow=0.6, delta=0.4）。

この分離により、「同じことを違うものにした」（flow平行 × delta直交）と「違うことを同じものにした」（flow直交 × delta平行）を区別できる。

---

## 2. 記憶は点ではなく領域

一つの記憶をベクトル1本に潰すと形が消える。「猫を撫でた → 安心した → 子供の頃を思い出した」は点ではなく、連する要素が作る集合。その集合は不定形。

形の表現：

- **合成記憶**: 似た記憶をグループ化して、重心だけじゃなく全メンバーを保持
- **edge/core分類**: グループ内で中心にいるか外縁にいるかを分ける。外縁が境界
- **多層ノイズ**: 動詞チェーンのパターンをノイズとして加える。バイアスの近似

---

## 3. 空間は異方的

**同じ経路でも出発点が違えば見え方が変わる。**

「猫を撫でた → 安心した」と「猫を撫でた → 不安になった」、違う方向。逆に「猫を撫でた → 安心した」と「雨音を聞いた → 安心した」、名詞も出発点も違うが似た形。


- **主成分軸**: 各合成記憶のSVD第一主成分を保存。クラスタが一番伸びてる方向。楕円体
- **異方的距離**: 軸方向の距離と軸に垂直な距離で重みが違う。軸方向に遠いのは「まだ中」、垂直方向に同じだけ遠いのは「外」
- **方向依存ノイズ**: 経験の多い方向には境界が柔らかく、未経験の方向には硬い

距離を非対称に。記憶Aから記憶Bへは近道があっても、BからAへは遠回りかもしれない。「猫から安心を思い出す」と「安心から猫を思い出す」が同じ強さとは限らない。連想に方向がある。

---

## 4. マルチモーダル拡張

**各モダリティは独立した空間を持ち、対応を前提にしない。**

CLIPやImageBindは大量のデータで「テキストと画像の対応」を事前に学習し、統一された空間を作る。このシステムはそれを前提としない。各モダリティ（テキスト/画像/音声）は独立したエンコーダと独立したベクトル空間を持ち、検索時に軸ごとに独立にスコアリングして重み付き合算する。

```
今:    similarity = 0.6 × cos(flow) + 0.4 × cos(delta)
拡張:  similarity = w1 × cos(flow) + w2 × cos(delta) + w3 × cos(visual) + w4 × cos(audio)
```

モダリティ間の橋渡しにはベクトル空間の統一ではなく、**バイアスノイズ**と**バウンダリー**の相互依存を使う。

### バイアスによる橋渡し

テンプレートバイアス（動詞チェーンの体験パターン）はモダリティに依存しない。「見る → 驚く → 調べる」という体験の骨格は、テキストでも画像でも共通。あるパターンがテキスト側で強いバイアスを育てたら、そのバイアスの強さ（テンプレートID + スカラー値）を画像側のバウンダリーにも適用できる。ベクトルの次元が違っても、バイアスの単位は次元非依存。

### バウンダリーによる橋渡し

合成記憶のedge/core分類をモダリティ横断的に影響させる。テキスト空間でedge（外縁）にいる記憶が、画像空間ではcore（中心）にいる場合、画像空間のcore情報がテキスト空間のedge判定を緩和する。各空間で独立に判定し、最終判定で統合する。

### 対応できないものは対応しない

別モダリティの軸はテキスト軸に完全には対応しない。対応できるものがあってもそれは偶然。このシステムは対応を前提にしないので、各軸が独自の情報を持ったまま残る。言葉にできない視覚の記憶は、言葉にできないまま保持される。

### 実装上の変更

既存のアーキテクチャを変える必要はない。

1. モダリティ別エンコーダの追加（CLIP, CLAP等）
2. embeddingsテーブルにカラム追加（visual_vector, audio_vector）
3. scoring.pyの重み付き合算に項を追加
4. consolidation.pyのバイアス適用をモダリティ横断に拡張
